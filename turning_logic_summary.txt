Turning Logic Summary for Traffic Simulation
=============================================

This document summarizes the circular turning logic used in the Traffic-simulation codebase. 
It can be used as a reference for implementing similar logic in other systems.

1. Core Concepts
----------------
- Approach: The direction from which a vehicle enters or exits an intersection (NORTH, SOUTH, EAST, WEST).
- Turn Type: Classified as LEFT_TURN, RIGHT_TURN, or STRAIGHT.
- Movement States: The vehicle transitions through STRAIGHT -> TURNING -> AFTER_TURN.

2. Determining Turn Type
------------------------
Based on the Entry Approach and Exit Approach:
- LEFT_TURN: (N->W), (W->S), (S->E), (E->N)
- RIGHT_TURN: (N->E), (E->S), (S->W), (W->N)
- STRAIGHT: Entry == Exit (or opposite, depending on road layout)

3. Geometry & Constants
-----------------------
The logic assumes a central intersection point (icx, icy) where roads meet.
- Road Width: W
- Half Width: H = W / 2
- Lane Offset: L (e.g., center of the lane relative to the road's midline)

Turn Radii:
- Left Turn Radius (outer arc): R_left = H + L
- Right Turn Radius (inner arc): R_right = H - L

4. Initialization (The "Prepare Turn" Phase)
--------------------------------------------
When a vehicle reaches the turn-trigger coordinate, it calculates:
- Turn Center (cx, cy): The pivot point for the circular arc.
- Start Angle (theta_start): Initial angle on the unit circle.
- End Angle (theta_end): Target angle on the unit circle.

Example (Entry from EAST):
- Left Turn: Center = (icx - H, icy - H), Start = 1.5*PI, End = 2*PI
- Right Turn: Center = (icx - H, icy + H), Start = 0.5*PI, End = 0

5. Execution (The "Update Turn" Phase)
--------------------------------------
The turn is performed by interpolating the angle based on linear speed.

Formula for Angle Increment:
   d_theta = (current_speed / radius)
   theta_new = theta_old + (direction * d_theta)
   (where direction is 1 for counter-clockwise, -1 for clockwise)

Formula for Position Update:
   X = cx + radius * cos(theta_new)
   Y = cy - radius * sin(theta_new)  // Note: Y is usually inverted in screen coordinates

6. Logic Flow
-------------
1. Move Straight: increment X or Y based on speed.
2. Check Trigger: If position passes `road.turn` coordinate, switch state to TURNING.
3. Prepare: Calculate cx, cy, start_angle, end_angle, and radius.
4. Update: 
   - Increment angle.
   - Check if angle reached `end_angle`.
   - Update X, Y using trig functions.
5. Exit: Once finished, set state to AFTER_TURN and update current `Approach` to `Exit Approach`.

7. Key Dependencies
-------------------
- Coordinate system where Y increases downwards (screen space).
- Road geometry providing the entry/exit relationship and intersection center.
